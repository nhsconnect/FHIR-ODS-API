{{include.para}} patient

See reference for details on this parameter. The patient parameter can be used two ways:

GET [baseUrl]/{{include.content}}?patient=[id]

This SHALL be supported for all endpoint provider systems.

id is the logical id of the patient on the server which is obtained by a Patient resource query.

id SHALL be the national identifier (NHS Number) for national systems. 

GET [baseUrl]/{{include.content}}?patient=42

Return all {{include.content}} resources for Patients with a logical id of 42
GET [baseUrl]/{{include.content}}?patient.identifier=[system]|[code]

This SHOULD be supported for all non national provider systems for queries by national identifiers (NHS Number).

This SHALL be supported for all intermediary (e.g. Trust Integration Engines or other middleware) systems.

system is a uniform resource identifier which defines which CodeSystem the identifer belongs to. For NHS Number this would be https://fhir.nhs.uk/Id/nhs-number and code would be the NHS Number.

GET [baseUrl]/{{include.content}}?patient.identifier=https://fhir.nhs.uk/Id/nhs-number|9876543210

Return all {{include.content}} resources for Patients with a NHS Number of 9876543210
